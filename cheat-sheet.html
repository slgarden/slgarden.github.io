CSS Positioning Cheat Sheet

The CSS positioning properties allow you to position an element. It can make elements overlap, send one back and bring another forward, and adjusts them depending on browser size. Elements can be positioned using the top, bottom, left, and right properties. However, these properties will not work unless the position property is set first. They also work differently depending on the positioning method. There are four different positioning methods: static, relative, fixed, and absolute.

Static Positioning

By default, HTML elements have static positioning--they are positioned according to the normal flow of the page. Static positioned elements are not affected by the top, bottom, left, and right properties. 

.static{
  background-color: green;
  width: 100%;
  padding: 5px;
}

This is a static box, which is the default position for elements on a page.

Relative Positioning

A relative positioned element is positioned based upon where it would otherwise originally appear. Its content can be moved and overlap other elements, but its space is still preserved. Relatively positioned elements are often used as parent blocks for absolutely positioned elements. The absolute element would be referred to as the "child". 

.relative1{
  background-color: yellow;
  padding: 5px;
}

.relative2{
  background-color: orange;
  width: 55%;
  padding: 5px;
  left: 25%;
  position: relative;
}

This is the first relative box. It is the same as the static box, and will only change if you add more properties.
This is the second relative box. It has different top, right, bottom, and left properties based upon the position of the first box.

Fixed Positioning

An element with a fixed position is positioned relative to the browser window, and will not move even if the window is scrolled. Fixed positioned elements are removed from the normal flow. The other elements in the browser are not affected by the fixed element. Fixed positioned elements can overlap other elements. 

.fixed{
  background-color: red;
  width: 20%;
  padding: 5px;
  position: fixed;
}

This is a fixed box. It doesn't move, even if you scroll!

Absolute Positioning

An absolute position element, aka the "child", is positioned based on the first non-static parent element. If there isn't a non-static parent element, the containing block is the HTML document. Absolutely positioned elements are removed from the normal flow. The document and other elements behave like the absolutely positioned element does not exist. Absolutely positioned elements can overlap other elements. 

.relative{
  background-color: blue;
  width: 100%;
  padding: 5px;
  position: relative;
}

.absolute{
  background-color: purple;
  width: 70%;
  padding: 5px;
  position: absolute;
  left: 20%;
  top: 120%;
}

This is a relative box. It is the parent of the absolute box below (aka its child).
This is an absolute box. It is the child of the relative box above. An absolute box will take the first non-static element as its parent.

Putting it All Together 

.container{
  position: relative;
  background-color: yellow;
  height: 200px;
}

.nav{
  position: absolute;
  background-color: purple;
  width: 15%;
  height: 155px;
}

.section{
  background-color: green;
  margin-left: 15%;
  height: 50px;
  border: white 1px dashed;
}

.footer{
  position: fixed;
  background-color: red;
  bottom: 0;
  left: 70px;
  height: 50px;
}

The container box is relative. It is the parent of the nav box.
The nav box is absolute based on the container. It is the child of the container.
The section box is static by default.
The section box is static by default.
The section box is static by default.
The footer box uses fixed positioning.
