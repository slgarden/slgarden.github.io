Blocks, Procs, & Lambdas

Things that Rhyme (sorta...)

2 February 2015

This week I've been focusing my Ruby knowledge on understanding the basic concepts of object-oriented programming and design. One incredibly useful facet of the Ruby language is how it deals with the computer science concept of closures. Before I dig into how Ruby distinguishes types of closures, let's first address what should be a nagging question: What on earth is a "closure"?!

In computer science, a closure is defined as a function that is contained within its own referencing environment. It includes variables that are bound to the environment of the closure. Ruby has several examples of closures, including Blocks, Procs, and Lambdas. These are listed in increasing order of functionality; in other words, lambdas can do more than procs, and procs have slightly more power than blocks. But let's start with the simplest first.

Even if you may not know exactly what a block is, if you have even the smallest familiarity with Ruby methods, chances are you've seen one before. A block, in essence, is a chunk of code that serves some specific function. What makes it different from a method, then? Well, it can't exist outside of a method. It simply serves as a way to evaluate code within a method. Here's a quick example: 

def doubled
numbers = [1,2,3,4]
numbers.each do |num|
num * 2
end
end
In the above code, the block is the code contained within the do..end keywords. Blocks may also be contained within { }, which is just a shorthand notation for do..end. As shown above, blocks simply help modify certain Ruby objects when executed within a method, in this case the #each method. Other methods that use blocks are #select, #collect, and #map, to name a few.

Procs act similarly to blocks, except they do one more thing: they can act as variables. Imagine you're writing a program that uses the same block within multiple method calls. Rather than rewriting that block over and over again, you can create a Proc, which stores the code of that block and can be called within your method instead of the block. For example: 

doubled_proc = Proc.new {|num| num * 2}

def doubled
numbers = [1,2,3,4]
numbers.each {|num| doubled_proc}
end

Lastly, we have Lambdas. This type of closure is the least common, but also has the greatest functionality. A lambda acts similar to a real method because you can use the return function within it. In addition, they are quite strict about arguments: they require that you pass the correct number of arguments through them (kinda like a method). Let's see a quick example:

lambda do |num|
puts num
return num
end
Notice the distinction between a lambda and an actual method is that there is no receiver of the given message.

As you can see, Ruby provides a lot of flexibility when it comes to closures. There are even more types of closures, including Methods with a capital M, but those objects lay beyond the scope of this post. Blocks, procs, and lambdas alone give us plenty of new information to digest as it is!
