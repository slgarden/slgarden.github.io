Ruby Hashes vs. JavaScript Objects

9 February 2015

After working with Ruby for at least four weeks, transitioning to JavaScript was a bit of a challenge. Most of the difficulty, naturally, can be attributed to learning a new way of doing something after growing accustomed to a different approach. JavaScript also comes with added complexity because many of the built-in methods that exist in Ruby are not available in JavaScript (meaning you have to build them from scratch based upon a limited set of built-in methods). Aside from the pros and cons of each language and the difficulties of transitioning between the two, I have noticed a huge amount of similarities between both languages. In particular, I'd like to delve into the likeness between Ruby's hashes and JavaScript's objects.

First and foremost, let's start off with some basic examples. I covered Ruby's arrays and hashes in a previous post a few weeks ago, but let me give a quick refresher. A hash is a data structure in Ruby that can be used to store, organize, and manipulate data. The following is an example of a way to present different sunglasses and their prices:

sunglasses = {"Rayban Wayfarer" => 155, "Warby Parker Haskell" => 95,
  "Dior So Real" => 450}

As you can see, hashes are represented by {} brackets, key/value pairs are separated by commas, and each key and respective value is separated by the => symbol. In JavaScript, objects are similar representations of data. They are constructed as follows:

var sunglasses = {raybanWayfarer: 155, warbyParkerHaskell: 95,
  diorSoReal: 450}

The above example looks very similar to the Ruby hash, except that the variable sunglasses needs to be declared with the keyword "var" and the keys and respective values are separated by a colon : instead of the => symbol. Additionally, the keys in JavaScript objects may also be referred to as properties. JavaScript keys/properties are more strict than those in Ruby hashes; they MUST be a string or symbol. Values in JavaScript, like in Ruby, can be any type of object: a string, a number/integer, an array, a boolean, or even another object.

Ruby and JavaScript also use similar methods to access keys and their values. In Ruby, the following code will return the price of Warby Parker Haskell sunglasses:

p "Warby Parker Haskell sunglasses are $"
  + sunglasses["Warby Parker Haskell"] + "."
# => "Warby Parker Haskell sunglasses are $95."

There are two methods to access values of properties in JavaScript objects. The first method is called dot notation. It is constructed like this:

console.log("Warby Parker Haskell sunglasses are $"
  + sunglasses.warbyParkerHaskell + ".");
// => "Warby Parker Haskell sunglasses are $95."

The alternative method to access values of properties in JavaScript is called bracket notation, which looks very similar to the notation used to access values of Ruby hashes.

console.log("Warby Parker Haskell sunglasses are $"
  + sunglasses["warbyParkerHaskell"] + ".");
// => "Warby Parker Haskell sunglasses are $95."

The difference between these two methods is that the first does not need the "" to denote a string; JavaScript automatically reads it as a string. Bracket notation, on the other hand, requires the "" for string properties, and this is because it also allows you to store the property name as a variable and use the variable name inside the brackets instead. This cannot be accomplished using dot notation.

All in all, Ruby hashes and JavaScript objects have key similarities and differences. Both are quite versatile objects, and are great for storing data!
